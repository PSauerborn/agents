# PROC-003 - Full Flow

## Flow Overview

**Overview**: Standard flow used to handle medium tasks. Generates plans, and task documents. Runs unit and integration tests.

**Scale**: Large tasks (6+ files modified) with architectural changes.

**Required Subagents**: `work-planner`, `task-decomposer`, `task-executor`, `quality-controller`, `risk-analyzer`, `risk-reviewer`

## Workflow

| Step | Agent | Purpose | Output |
| ------ | ------- | --------- | -------- |
| 1 | work-planner | Analyses requirements from `requirements-analyzer` and generates a work plan plan. | Work plan, written to the `documentation-criteria` canonical work-plan location. |
| 2 | risk-analyzer | Analyzes work plan and perform independent risk review> | Risk plan, written to the `documentation-criteria` canonical risk-plan location. |
| 3 | (orchestrator) | Wait for user approval of work and risk plan generated by `work-planner` and `risk-reviewer`. **[STOP]** | |
| 4 | task-decomposer | Decompose work plan into executable task files for execution. | A series of executable task files, written to the `documentation-criteria` canonical task location. |
| 5 | task-executor | Execute task file(s). | |
| 6 | quality-controller | Review code changes for adherence to standards, and generate task file for remediation (if required). | Quality report and (if required) a remediation task file, each written to its `documentation-criteria` canonical location. |
| 7 | (orchestrator) | Determine if qc remediation execution is required based on output of `quality-controller` agent output. Skip step 7 if no remediation is required. | |
| 8 | task-executor | Execute the quality-control remediation task (at its `documentation-criteria` canonical location) if present. | |
| 9 | risk-reviewer | Reviews risk plan, and identifies risk remediation tasks (if required). | Risk remediation executable task file written to its `documentation-criteria` canonical location. |
| 10 | (orchestrator) | Determine if risk remediation execution is required based on output of `risk-reviewer` agent output. Skip step 11 if no remediation is required. | |
| 11 | task-executor | Execute the risk remediation task (at its `documentation-criteria` canonical location) if present. | |
| 12 | documenter | Update documentation. | Changeset written to the `documentation-criteria` canonical changeset location. |

## Additional Instructions

* Parallelize execution where possible - each task file has a `Task Dependencies` section that details any other tasks that it depends on. Run multiple, parallel instances of the `task-executor` agent where possible.
